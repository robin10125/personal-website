Fundamentals of JAX Part 1

June 18, 2026

JAX is all about functions, rather than objects holding state. It's syntax and form mirrors that of numpy, and uses composable function transformations. In JAX, arrays are immutable. Transformations create new ararys. During compilation, if the compiler determines that old values of an array are not used, copying is turned into in place writing. Functions in JAX are "pure", meaning that everything that is not constant must enter the funciton as an argument, and everything a function produces must leave as a return value. Randomess is also different in JAX than PyTorch. When gerneating random numbers in JAX, first you generate a key, then pass the key into the random generator. The generated values are deterministic, so every generation from the same key is the same. You can split keys to get multiple keys if you need multiple random values.
